Model Context Protocol

A protocol for agentic computing based on JSON-RPC 2.0.

by Charles Iliya Krempeaux (reiver)

Model Context Protocol (MCP) is a JSON-RPC 2.0 based protocol designed to be used with Agentic Computing.

Model Context Protocol is a type of universal plug and socket that allows agents to "see" and affect the outside (digital or physical) world.

You can think of MCP as a type of API that agents use to talk to tools (and vice versa).

An agent can read your e-mail inbox using MCP. An agent can also send an e-mail message using MCP. An agent can make a purchase on your behalf using MCP. An agent can get the reported temperature in a city using MCP. And, in the future, an agent may activate or control one or more robots using MCP. Etc.

Becoming An Agent

MCP is often what enables LLMs to affect the world. And thus, is an big part of what turns a LLM into an agent.

(An LLM isn't an agent if it cannot affect the (virtual or physical, etc) world.
